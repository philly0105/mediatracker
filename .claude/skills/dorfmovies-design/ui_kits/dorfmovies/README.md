# DorfMovies — UI Kit

A high-fidelity, click-through recreation of the **DorfMovies** app, composed from
this design system's component primitives (`window.DorfMoviesDesignSystem_f30e74`).

## Run
Open `index.html`. It loads React + Babel + Lucide + the compiled `_ds_bundle.js`, then
mounts the screen files. (The bundle is generated by the design-system compiler at the
project root.)

## Screens (interactive)
- **Dashboard** (`DashboardScreen.jsx`) — eyebrow + gradient hero, bento stat tiles
  (`SpotlightCard` + `StatTile`), a "now watching / Live" card with a blurred poster
  bleed, and the "Recently Watched" `PosterCard` grid.
- **Search** (`SearchScreen.jsx`) — live-filtering pill `Input`; results as rows with
  `Watched` / `Watchlist` status `Badge`s.
- **Library** (`LibraryScreen.jsx`) — Movies / Shows / Watchlist as `MediaRow` grids with
  editable star ratings.
- **Detail modal** (`DetailModal.jsx`) — poster + meta + genre chips + review, with
  "Add to Watchlist" / "Mark as Watched" actions that fire a toast.
- **Shell** (`Sidebar.jsx`, `App.jsx`) — fixed 256px nav rail, ambient orb background,
  view switching, and the toast.

## Interactions to try
- Click any nav item to switch views.
- Type in **Search** to filter; click a result to open the detail modal.
- In the modal, **Add to Watchlist** / **Mark as Watched** → toast + status updates.
- Hover posters (zoom + glow) and rate titles in the library (stars persist in session).

## Data
`data.js` exposes `window.DORF_DATA` — a small curated set of titles with TMDB poster
URLs. Poster images fall back to a tinted title well if a URL fails, so the kit always
renders cleanly offline.

> Not recreated (shown as a placeholder): Calendar, Collections, Recommendations, Versus,
> Stats, Settings — these exist in the full product but are out of scope for this kit.
