# Design Spec: DorfMovies Autumn Pine Reskin

## 1. Summary
Migrate the mediatracker application UI from its current violet/orange/rose glassmorphic design to the **Autumn Pine** earthy theme defined in the DorfMovies design system (`.agents/skills/dorfmovies-design/`). This includes replacing hardcoded styling tokens, porting components/primitives, updating visual spacing and radii, and adapting existing pages.

## 2. Goals
* **Exact fidelity, not a recolor.** Every page in the app should look and feel like it was built from the DorfMovies design system — exact radii, shadows, spacing, motion curves, and component patterns, not just the right hue applied to the old markup.
* Update the application canvas to use the warm brown-black background (`#100e09` / `--bg-void`) with the `--page-radial` gradient.
* Standardize on the Outfit font family.
* Port every primitive (`Button`, `Badge`, `Eyebrow`, `IconChip`, `Input`, `NavItem`, `PosterCard`, `MediaRow`, `StarRating`, `StatTile`, `GlassCard`) into `components/ui/` as real `.tsx`, and use them everywhere instead of one-off Tailwind utility soup.
* For the four screens the design system explicitly mocked up (Dashboard, Search, Library, Sidebar/nav, Detail modal — `ui_kits/dorfmovies/*.jsx`), match that mockup's structure directly.
* For every other page (movies, shows, watchlist, collections, stats, calendar, versus, settings, login, lists, import, person, recommendations, share) — which have no mockup to copy — rebuild them using the same ported primitives and token scale so they're indistinguishable in style from the mocked screens, even without a literal reference.

## 3. Architecture & Styling Integration

### 3.1 Token import
Copy the design system's token stylesheets into the app itself as `app/styles/dorfmovies-tokens.css` (merging `tokens/colors.css`, `typography.css`, `spacing.css`, `effects.css`, `textures.css`, `keyframes.css`, `fonts.css`), rather than importing directly from `.agents/skills/dorfmovies-design/`. That folder is agent-facing reference material that can be regenerated by a future skill/design-sync run; production CSS shouldn't have a build dependency on it. `app/globals.css` will `@import` the copied file, registering the custom properties on `:root`.

### 3.2 Primitive library (the primary mechanism for exact fidelity)
A Tailwind color-only remap (overriding `--color-violet-*` etc. so existing classes resolve to new hex values) was considered as a shortcut, but it only fixes hue — pages would keep the old rounded-full pills, rounded-2xl cards, backdrop-blur glassmorphism, and scale-on-hover motion, which is explicitly *not* what the design system specifies (flat opaque surfaces, 4–12px editorial radii, lift-not-scale hover, no blur). Since the goal is an exact match, every component is rebuilt on the real ported primitives instead:

| Primitive | Source | Destination |
|---|---|---|
| `Button` | `components/core/Button.jsx` | `components/ui/Button.tsx` |
| `Badge` | `components/core/Badge.jsx` | `components/ui/Badge.tsx` |
| `Eyebrow` | `components/core/Eyebrow.jsx` | `components/ui/Eyebrow.tsx` |
| `IconChip` | `components/core/IconChip.jsx` | `components/ui/IconChip.tsx` |
| `Input` | `components/core/Input.jsx` | `components/ui/Input.tsx` |
| `NavItem` | `components/navigation/NavItem.jsx` | `components/ui/NavItem.tsx` |
| `PosterCard` | `components/media/PosterCard.jsx` | `components/ui/PosterCard.tsx` |
| `MediaRow` | `components/media/MediaRow.jsx` | `components/ui/MediaRow.tsx` |
| `StarRating` | `components/media/StarRating.jsx` | merged into `components/RatingStars.tsx` |
| `StatTile` | `components/surfaces/StatTile.jsx` | `components/ui/StatTile.tsx` |
| `GlassCard` (now solid) | `components/surfaces/GlassCard.jsx` | `components/ui/Card.tsx` |
| `SpotlightCard` | `components/surfaces/SpotlightCard.jsx` | update existing `components/ui/SpotlightCard.tsx` in place to use new tokens |

Every page rewrite in Section 4 composes from this table rather than hand-rolling new Tailwind classes per page.

### 3.3 Tailwind utility cleanup (mop-up only)
Anything that can't reasonably route through a ported primitive (one-off literal hexes like `bg-[#09090B]`, `#F97316`, bare `text-white` / `border-white/*` neutral-chrome usage) gets replaced directly with the matching token (`var(--bg-base)`, `var(--brand-mark)`, `var(--text-primary)`, `var(--border-subtle)`, etc.) file by file. No `@theme` color-scale override — it would mask leftover unconverted markup instead of forcing it to be fixed.

### 3.4 Custom Radii Mapping
The new spacing scheme uses tighter, editorial corner radii:
* Buttons, Inputs, Chips: `var(--radius-sm)` (4px)
* Nav Items, Rows: `var(--radius-md)` (6px)
* Standard Cards: `var(--radius-lg)` (8px)
* Poster Cards: `var(--radius-xl)` (10px)
* Modals, Popovers: `var(--radius-2xl)` (12px)
* Eyebrows, Badges: `var(--radius-pill)` (9999px)

### 3.5 Custom Shadows & Glows
* Standard Card Shadows: `var(--shadow-md)`
* Poster Shadow: `var(--shadow-poster)`
* Accent Glow: `var(--glow-violet)` — despite the legacy name this resolves to a deep shadow plus a thin **amber/gold** ring (`rgba(216, 166, 78, 0.30)`), not pine green. Use `var(--glow-violet)` as-is for the default hover ring (matches the design system's own `GlassCard` default); reserve `var(--glow-live)` (rust ring) only for "live/now-watching" treatments.

## 4. Component Refactoring Specifications

### 4.0 Mockup-to-page mapping
The download's `ui_kits/dorfmovies/` directory contains five reference mockups built against fake static data (`data.js`). Where a real page maps to one, that mockup's structure (layout, spacing, composition — not its literal data) is the direct reference:

| Mockup | Real page(s) |
|---|---|
| `DashboardScreen.jsx` | `app/page.tsx` (+ `components/DashboardRecentCards.tsx`) |
| `SearchScreen.jsx` | `app/search/page.tsx` |
| `LibraryScreen.jsx` | `app/movies/page.tsx`, `app/shows/page.tsx`, `app/watchlist/page.tsx`, `app/collections/page.tsx`, `app/lists/page.tsx` (same grid-of-cards pattern) |
| `Sidebar.jsx` | `components/Sidebar.tsx` |
| `DetailModal.jsx` | `components/MediaInfoModal.tsx` |

Pages with no mockup counterpart — `app/stats`, `app/calendar`, `app/versus`, `app/settings`, `app/login`, `app/import`, `app/person/[name]`, `app/recommendations`, `app/share/*`, `app/show/[id]`, `app/collections/[id]`, `app/lists/[id]` — are rebuilt from the ported primitive table in 3.2 directly: page headers use the `Eyebrow` + display-type heading pattern from `DashboardScreen.jsx`, stat numbers use `StatTile`, lists of items use `MediaRow`/`PosterCard`, forms use `Input`/`Button`, and surfaces use `Card`. No new visual vocabulary is invented for these pages — they reuse exactly what's already defined for the mocked screens.

### 4.1 Root Layout (`app/layout.tsx`)
* Keep the Google `Outfit` font configuration.
* Change body background class to use `bg-[var(--surface-page)]` with the `bg-[var(--page-radial)]` background image.
* **Recolor, don't remove**, the three floating gradient orb elements. The design system explicitly calls for "three large ambient warmth orbs behind content (pine top-left, rust bottom-right, amber center — very low opacity, 120–150px blur)" and ships dedicated tokens for them (`--orb-violet`, `--orb-rose`, `--orb-orange`). Swap each orb's `rgba(...)` literal for the matching `--orb-*` token and adjust position/size to match (pine top-left, rust bottom-right, amber center) instead of deleting the elements.

### 4.2 Navigation Sidebar (`components/Sidebar.tsx`)
* Desktop aside container background set to `var(--bg-base)`, right border to `1px solid var(--border-subtle)`.
* DorfMovies branding text styled in Outfit bold/extrabold. "Dorf" is warm cream (`var(--text-primary)`, not `var(--white)`), "Movies" is pine green (`var(--brand-mark)`) — confirmed against `assets/wordmark.html`.
* Nav items styled to match the `NavItem` primitive:
  - Active item: background `rgba(255,255,255,0.05)`, border `1px solid var(--border-default)`. Accent highlight in green (`var(--accent)`).
  - Hover item: text brightens to `var(--zinc-300)`, icon scales `scale(1.1)`.
* User profile container: background `rgba(255,255,255,0.01)`, border `1px solid var(--border-faint)`, radius `var(--radius-md)`.
* Update mobile bottom bar and slide-out more drawer with identical tokens (`var(--bg-base)`, `var(--border-subtle)`).

### 4.3 Horizontal Media Card (`components/MediaCard.tsx`)
* Reskin the card wrapper to match `MediaRow.jsx`:
  - Background: `var(--glass-card)` (normal), `var(--glass-card-hover)` (hover).
  - Border: `1px solid var(--border-subtle)` (normal), `1px solid var(--border-strong)` (hover).
  - Radius: `var(--radius-lg)`.
  - Box shadow: none (normal), `var(--glow-violet), var(--inset-hairline)` (hover).
  - Hover motion: `translateY(-2px) scale(1.01)` using `var(--dur-base)` transition.
* Rating element uses `StarRating` rules (see Section 4.5).
* Badges (TV / Movie labels) styled as pill badges with neutral tints (`var(--border-faint)`, background `rgba(255,255,255,0.05)`).
* Delete and edit buttons styled with ghost button states.

### 4.4 Vertical Poster Card (`components/CollectionMovieCard.tsx`)
* Wrapper styled to match `PosterCard.jsx`:
  - Border: `1px solid var(--border-subtle)` (normal), `1px solid var(--border-default)` (hover).
  - Radius: `var(--radius-xl)`.
  - Background: `var(--bg-void)`.
  - Shadow: `var(--shadow-poster)` (normal), `var(--glow-violet), var(--shadow-md)` (hover).
  - Hover motion: `var(--lift-poster)` (`scale(1.02)`) using `var(--dur-slow)`.
* Poster Image: aspect ratio `2 / 3`. Hover scales image (`scale(1.10) rotate(1deg)`) with transition `var(--dur-slower)`.
* Hover overlay: linear gradient overlay with text in white.
* Badges: replace current green/purple indicator icons with semantic pill-badges (Teal badge for watched, Amber badge for watchlisted).

### 4.5 Star Rating Component (`components/RatingStars.tsx`)
* Unfilled star track: `var(--zinc-700)`.
* Filled star/hover state color: `--amber-400` / `--rating`.
* Retain half-star selection mapping and click listeners.

### 4.6 Form Elements & Modals (`components/EditEntryModal.tsx`, `components/MediaInfoModal.tsx`)
* Modal wrapper: background `var(--surface-modal)`, border `1px solid var(--border-subtle)`, radius `var(--radius-2xl)`, shadow `var(--shadow-xl)`.
* Text Inputs / Search Inputs: background `var(--surface-input)`, radius `var(--radius-sm)`, border `1px solid var(--border-default)` (focus `1px solid var(--border-focus)`).
* Primary buttons (e.g. Save, Add): pill/button styled with `--btn-primary-bg` (pine green) and text `--btn-primary-fg` (deep slate-green).
* Ghost buttons (Cancel, Close): background `--btn-ghost-bg`, border `1px solid var(--border-default)`.

### 4.7 Remaining pages and components (no direct mockup)
Applies the same primitive table (3.2) and token scale (3.4–3.5) to everything not already covered above, with no page-specific visual decisions left open:
* **Pages:** `stats`, `calendar`, `versus`, `settings`, `login`, `import`, `person/[name]`, `recommendations`, `share/*`, `show/[id]`, `collections/[id]`, `lists/[id]`.
* **Components:** `SimilarModal.tsx`, `StatsCharts.tsx` (recharts theme colors switch to `--green-500`/`--amber-400`/`--rust-400`/`--teal-400`), `CollectionMovieCard.tsx` is covered by 4.4, `BackButton.tsx`, `ShareToggle.tsx`, `SelectableOverlay.tsx`, `PopularCollectionsFeed.tsx`, `PasswordChangeForm.tsx`, `EpisodeTracker.tsx`, `MultiSelectProvider.tsx`, `BentoGrid.tsx`.
* Any literal hardcoded hex left in these files (see 3.3) is replaced with its matching token, not left as-is.

## 5. Verification Plan
* Validate compilation of CSS variables inside the Next.js bundle (`npm run build` / dev server boots clean).
* Visually check every route in the app (not just the four mocked screens) against the design system's token scale: canvas color, card surfaces, radii, shadows, and font.
* Verify Sidebar layout rendering on desktop and mobile resolutions.
* Verify hover states and motion (lift, not scale, on cards) on MediaCard, CollectionMovieCard, and StatTile-based dashboard tiles.
* Verify star rating inputs and modal/popover forms styling.
* Grep the diff for leftover `violet|rose|orange|emerald|amber|zinc`-named Tailwind utilities and literal hex colors outside the token file — zero should remain.
